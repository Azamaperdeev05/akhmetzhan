# Phase 5 Defense Presentation Script (5-7 minutes)

## 1) Кіріспе (40-60 сек)

Сәлеметсіздер ме, құрметті комиссия мүшелері.  
Менің дипломдық жобамның тақырыбы: **PhishGuard — корпоративті поштадағы фишинг хабарламаларды автоматты анықтау модулі**.

Жобаның мақсаты: Gmail негізіндегі корпоративті ортада келетін хаттарды автоматты талдап, фишинг тәуекелін ерте анықтау және әкімшілік бақылауды жеңілдету.

Негізгі құндылығы:

- адам факторын азайту;
- қауіп туралы жедел белгі беру;
- талдау нәтижелерін dashboard арқылы түсінікті көрсету.

## 2) Мәселе және шешім идеясы (50-70 сек)

Практикада классикалық спам-фильтрлер жаңа шаблондағы фишинг хаттарды әрқашан дәл ұстай бермейді.  
Сонымен бірге, қызметкер деңгейінде жалған хатты бірден ажырату қиын.

Менің шешімім үш қабатты:

1. **Модельдік талдау** — хат мәтінінен фишинг ықтималдығын есептеу.
2. **Қосымша ережелік талдау** — URL, SPF/DKIM, header сигналдары.
3. **Операциялық қабат** — базаға сақтау, label қою, dashboard-та визуализация.

## 3) Архитектура (60-90 сек)

Жүйе ағыны:

1. Gmail API арқылы inbox-тен хат алынады.
2. Хат мәтіні preprocess жасалады.
3. Модель фишинг ықтималдығын береді.
4. URL және header анализі қосылады.
5. Қорытынды label/risk есептеледі.
6. Нәтиже SQLite базасына жазылады.
7. Dashboard `/`, `/emails`, `/stats`, `/settings` беттерінде көрсетіледі.

Техникалық стек:

- Python, Flask, SQLAlchemy
- Transformers/PyTorch
- Gmail API OAuth2

## 4) Іске асырылған фазалар (60-90 сек)

Жоба 5 фаза бойынша жүргізілді:

1. **Фаза 1** — дерек өңдеу және baseline/BERT training pipeline.
2. **Фаза 2** — analyzer + DB интеграциясы.
3. **Фаза 3** — Gmail runtime, retry, dedup, label flow.
4. **Фаза 4** — dashboard және end-to-end ағын.
5. **Фаза 5** — quality gate, automated reports, defense artifacts.

Нақты валидациялар:

- test suite толық жасыл: `15 passed`;
- Phase 3 real Gmail run: `target_count=20` критерийі орындалды, `scanned_total=82`;
- Phase 4 E2E: барлық dashboard endpoint `200 OK`.

## 5) Live Demo сценарийі (2-3 минут)

## Демо мақсаты

Inbox-тен алынған хаттардың базаға түсіп, dashboard-та визуалданатынын көрсету.

## Командалар реті

1. Scanner бір цикл:

```bash
python main.py --once
```

2. Phase 4 E2E қысқа валидация:

```bash
python scripts/run_phase4_e2e_demo.py --db-url sqlite:///phase4_demo.db
```

3. Dashboard ашу:

```bash
python dashboard/app.py
```

Ашу: `http://localhost:5000`

## Демо кезінде көрсетілетін беттер

1. `/` — жалпы summary және соңғы scan жазбалары.
2. `/emails` — хаттар тізімі, risk/probability/SPF/DKIM.
3. `/stats` — соңғы кезең статистикасы графигі.
4. `/settings` — `PHISHING_THRESHOLD`, `SCAN_INTERVAL_MINUTES` басқару.

## 6) Нәтиже және шектеулер (40-60 сек)

Нәтиже:

- толық жұмыс істейтін MVP бар;
- real Gmail интеграциясы жұмыс істейді;
- сапа бақылауы мен қорғанысқа дайын артефакттар автоматтандырылған.

Шектеу:

- ағымдағы метрикалар шағын/демо датасет әсерін көрсетеді;
- финал ғылыми қорытынды үшін толық Kaggle датасетпен ұзақ training қажет.

## 7) Қорытынды (20-30 сек)

PhishGuard корпоративті пошта қауіпсіздігін арттыруға бағытталған практикалық шешім ұсынады:  
**автоматты талдау, бақылау, және интерпретациясы бар end-to-end модуль**.

## 8) Комиссия сұрақтарына қысқа жауаптар

## Q1: Неге дәл BERT?

Өйткені контексті жақсы түсінеді, классикалық bag-of-words тәсілдеріне қарағанда фишингтегі тілдік паттерндерді сапалы ұстайды.

## Q2: Неге ережелік URL/header анализін қосу керек?

Модель жалғыз өзі жеткіліксіз болуы мүмкін. SPF/DKIM/URL сигналдары модельден тыс тәуекелді жабады, нәтижені түсіндіруге көмектеседі.

## Q3: Жүйе масштабтала ма?

Иә. Архитектура модульдік: fetch/analyze/store/ui бөлек. Келесі қадамда queue + worker + REST API арқылы горизонталь масштабтауға болады.

## Q4: Қате анықтау (false positive) қалай реттеледі?

Threshold пен risk логикасы бапталады (`/settings`). Сонымен қатар whitelist/feedback механизмін қосуға болады.

## Q5: Келесі даму бағыты?

Толық датасетпен қайта үйрету, көппошталық провайдерлер (Outlook/Yandex), және explainable alerts.
